# Bug Reports

Предусловие: открыта страница https://career.avito.com/vacancies/ со следующими применёнными фильтрами:

1) В поиске набрано «Аналитик»
2) В фильтре «Направление» выбрано «Data Science»
3) Выбран подфильтр «Поддержка пользователей»
4) В фильтре «Город» выбран «Санкт-Петербург»
5) В фильтре «Формат работы» выбран «Можно удалённо»
6) Включена галочку «Руководящая роль»


## BUG-1. Фильтр «Руководящая роль» показывает вакансии без руководящих ролей

Ожидаемый результат:

Фильтр «Руководящая роль» показывает вакансии только с руководящей ролью

Фактический результат:

Фильтр «Руководящая роль» показывает вакансии БЕЗ руководящих ролей

Серьёзность: Major

Приоритет: High. Ошибка нарушает основную логику фильтрации и мешает пользователю находить релевантные вакансии. Фильтр теряет смысл.


## BUG-2. С фильтром «Санкт-Петербург» отображаются вакансии из других городов

Ожидаемый результат:

При включенном фильтре «Санкт-Петербург» отображаются вакансии только из Санкт-Петербурга

Фактический результат:

При включенном фильтре «Санкт-Петербург» отображаются вакансии из других городов.

Серьёзность: Major

Приоритет: High. География один из ключевых фильтров. Пользователь ищет работу в конкретном городе.



## BUG-3. Фильтр «Можно удалённо» отображает вакансии с меткой «Офис».

Ожидаемый результат:

При включенном фильтре «Можно удалённо» отображаются только вакансии с меткой «удалённо».

Фактический результат:

При включенном фильтре «Можно удалённо» отображаются вакансии с меткой «офис».

Серьёзность: Major

Приоритет: High. Один из ключевых фильтров. Пользователь ищет удалённую работу.



## BUG-4. В фильтре «Команда» отображается «Вакансий пока нет», хотя поиск выдал вакансии.

Ожидаемый результат:

В фильтре «Команда» отображается список команд, если вакансии по заданным фильтрам есть.

Фактический результат:

В фильтре «Команда» отображается «Вакансий пока нет», хотя поиск выдал вакансии.

Серьёзность: Minor

Приоритет: low. Выдача всё-таки есть, поэтому фильтр не критичен. Больше UX-проблема.



## BUG-5. С фильтром «Поддержка пользователей» отображаются вакансии других категорий

Ожидаемый результат:

С фильтром «Поддержка пользователей» отображаются вакансии только в категории «Поддержка пользователей»

Фактический результат:

С фильтром «Поддержка пользователей» отображаются вакансии других категорий

Серьёзность: Major

Приоритет: Medium. Баг влияет на релевантность, но фильтр не основной для многих пользователей. 


## BUG-6. В поисковой выдаче отображается «Ничего не нашлось», хотя сверху и снизу есть вакансии.

Ожидаемый результат:

В поисковой выдаче отображается либо «Ничего не нашлось», либо вакансии.

Фактический результат:

В поисковой выдаче отображается «Ничего не нашлось», хотя сверху и снизу есть вакансии.

Серьёзность: Major

Приоритет: Medium. Выдача вакансий всё-же есть, но такая фаза сбивает с толку пользователя и может повлиять на решение продолжать поиск


## BUG-7. Счётчик вакансий в разделе «Клиентский сервис» не соответствует действительности

Ожидаемый результат:

Счётчик вакансий в разделе «Клиентский сервис» соответствует действительности

Фактический результат:

Счётчик вакансий в разделе «Клиентский сервис» не соответствует действительности: Счётчик показывает 2, хотя вакансий отображается 1

Серьёзность: Minor

Приоритет: low. Не ломает поиск, только UX-проблема.


## BUG-8. Рядом с поиском битый значок крестика для сброса набранного текста

Ожидаемый результат:

Рядом с поиском отображается крестик для сброса набранного текста

Фактический результат:

Рядом с поиском битый значок крестика для сброса набранного текста

Серьёзность: Trivial

Приоритет: low. Текст можно удалить другим способом, но битая иконка портит внешний вид
